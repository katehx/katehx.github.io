Autonomous Robot Code

Program: FIRST Tech Challenge - 2021-21 Ultimate Goal Season

Date: August 2020 - February 2021

Project Objective: Design, build, and program a robot to collect and shoot rings into appropriate goals both autonously and driver controlled. Additional game tasks include parking in appropriate locations, delivering moveable goals to appropriate locations, and hitting higher percision targets during special time periods.

Game Summary: https://www.youtube.com/watch?v=VVxz2Kr9Jgw

Languages & Libraries: Android Studio Java with RoadRunner Pathplanning Library, OpenCV Vision Sensing Library, and PID motor control.

Applied Skills & Concepts: Sensor Integration, PID Feedback Control Systems, Vision Processing, Path Planning Algorithms, & Code Optimization.

Performance Statistics:

  • 10th overall robot score in North Carolina for 2020-21 season
  • 7th ranked autonomus score at North Carolina State Championship
  • 149th ranked OPR internationally
GitHub Repository

Technology Used:

  • Vision Sensing: Used a USB camera and the OpenCV Vision Library to autonously read if there was a stack of 0, 1, or 4 rings at the beginning of the match. Based off of this random ring placement the robot would then follow a unique path for bonus points.
  • Dead-Wheel Encoders: Three dead-wheel encoders were mounted on the chassis of the robot. These encoders would directly read the robots position on the field and elimate the encoder drift or slippage that would occur with the motor encoders directly attached to the drive wheels.
  • RoadRunner Pathplanning: This libary enabled the data from the dead-wheel encoders to be used to create advanced movement paths. The useage of splines along with the dead-wheels allowed for consistent and effecient autonoums paths. The libary required the tuning of multiple PID feedback loops and multiple constraints to be tuned to the specific robot.
  • PID Shooter Feedback: The flywheel on the robot was prone to inconsistencies from battery voltage. Implementing and tuning a PID feedback loop onto this shooter helped the system get to a specific RPM rather than a percentage based power. The PID works to track and then correct the flywheel as it goes off track of the goal RPM.

Key Learnings & Takeaways: This project was my first experience integrating code that would interact with the physical world. It introduced a new perspective that came with significant challenges, but also taught me valuable lessons. Factors such as battery voltage, floor and wheel friction, and trajectory changes due to varying flywheel speeds were just a few of the obstacles I encountered. In these situations, the importance of sensors and more advanced algorithms became clear.

Dead-wheel encoders proved invaluable for tracking the robot's position on the field, even when slippage occurred with motor encoders from diving. Similarly, motor encoders on the shooter's flywheel, controlled by a PID feedback loop, ensured consistent ring trajectories despite inconsistencies in battery levels.

The implementation of libraries throughout this project highlighted the importance of being able to read and comprehend other people's code. Many issues during the implementation process arose from not fully understanding the purpose of functionality of the code. However, once I took the time to read through and understand the code, tuning it for my specific situation became much easier.

While working with the OpenCV Vision Library I had to familiarize myself with concepts such as Gaussian blur, which is used to reduce noise and smooth images for better vision processing. This comprehension was crucial for accurately reading and interpreting the vision tuning numbers. Additionally, I had to learn how the library handles adjustments of bounding box sizes and placement along with the hue range adjustments. Understanding how to properly use the library lead to consistent and reliable vision sensing.

Personal Task Manager

Course: ITCS 3112 - Design and Implementation of Object-Oriented Systems

Date: Fall 2024

Project Objective: Design and implement a small-scale software application using object-oriented programming principles.

Project Milestones:

  • Milestone 1: Topic Proposal
  • Milestone 2: Project Requirements Document & Use Case Diagram
  • Milestone 3: Class Diagram
  • Milestone 4: Activity Diagram

Languages & Libraries: Java with Java Swing library used for GUI.

Applied Skills & Concepts: Encapsulation, Inheritance, Abstraction, Polymorphism, List Sorting Implementation, User Input Management, GUI Creation, Software Development Life Cycle (SDLC), & UML Diagrams.

GitHub Repository

Key Learnings & Takeaways: Throughout this project, I applied several Object-Oriented Programming (OOP) principles to design and implement a personal task management system. Encapsulation was a key focus, ensuring that each class had a specific responsibility and that only the appropriate sections of code would be able to access the class data. This was achieved through the use of private fields and well-structured getter and setter methods. Inheritance helped to establish a clear hierarchy amongst classes, particularly in creating a student task manager that extended the general task manager. This approach eliminated redundant code and allowed me to concentrate on adding new and expanded functionalities. By applying these principles, along with others, I developed a system that was modular, maintainable, and easy to understand, aligning with the best practices in OOP.

In addition to the OOP coding principles practiced, this project served as a hands-on practice run of the Software Development Life Cycle (SDLC). I learned multiple approaches to defining project requirements, measuring success, and managing intergration of systems. Methodologies like Waterfall, SCRUM, and Agile showed the strengths of taking different approaches to project management depending on the project and the team working on it. Finally, I developed multiple UML diagrams for the system I was developing. These diagrams not only let me practice the diagram creation process, but often led to me viewing my system in a new light. I learned that UML documentaion not only makes it easy to communicate your system to others, but that it can also help you better understand your own system.
